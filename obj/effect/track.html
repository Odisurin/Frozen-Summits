track
/obj/effect/track

Vars

ambiguous_track_typeLike above, except what you get if you are not good.
creation_timeWhen this was created. Adjusts difficulty of locating / analyzing.
creatorThe exact mob that created this. Only used to see if the spotter can notice their own tracks (fairly easy)
deletion_timerThe timer handling deletion. Saved to potentially adjust it.
depthIf the depth of the tracks is abnormal, e.g. because of heavy armor.
facingThe way the mob was facing when this was created. Obviously can be messed with if you e.g. walk backwards.
known_byList of mobs aware of this track.
overwritesTracks how many tracks have been chain-overwritten before this track. Could indicate a commonly passed area.
real_icon_stateThe visible state for those that know this.
real_imageThe image knowers see.
special_movementIf the creator was moving in a special way, e.g. running / sneaking. Difficult to discern.
track_typeWhat kind of foot, or footwear, created this.
tracking_modifierSome things may be easier or harder to track. This adjusts the base difficulty accordingly.

Procs

add_knowerAdds a new person to the list of people who can see this track.
check_revealHandles checks for if a mob can reveal this. Also returns FALSE if already known to mob.
clear_creator_referenceClears the reference to the creator. Is replaced by the above proc if the creator analyzes it.
handle_revealingHandles revealing the track, including checking how well the tracker can analyze it.
remove_knowerRemoves a knower from the known ones. Usually only done when qdeleted.
track_expireCalled when the track's time expires, at which point it becomes indistinguishable (aka, deleted)
